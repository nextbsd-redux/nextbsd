# One pkg per line; lines starting with # are ignored.
# Build-only FreeBSD-* pkgbase packages — needed inside the chroot to
# compile the gershwin system-domain libraries (libdispatch + GNUstep
# stack), then purged before the slim/mkuzip pass so they don't ship
# in the ISO. Mirrors the buildpkgs.txt purge pattern but for pkgbase
# pkgs that base.txz used to drag in unconditionally.
#
# Installed alongside pkglist-base.txt in the same host-side
#   pkg -R $WORK/pkg-config -r $WORK/rootfs install -y ...
# call.

# clang/lld for cmake -DCMAKE_C_COMPILER=clang in the GNUstep chroot build.
FreeBSD-clang
FreeBSD-lld

# Base toolchain — make(1), ar, ranlib, etc.
FreeBSD-toolchain

# Headers. cmake/configure inside the chroot need /usr/include/* to
# compile libobjc2, libs-base, libs-corebase. clang itself needs its
# own headers (clang-dev).
FreeBSD-clang-dev
FreeBSD-runtime-dev
FreeBSD-clibs-dev
FreeBSD-libcompiler_rt-dev
FreeBSD-libcompat-dev
FreeBSD-libexecinfo-dev
# FreeBSD-pam-dev — dropped at PAM port iter 3 (issue #97).
# Project principle: never install pkgbase packages we replace
# with Apple-source code in the build chroot. Installing them
# pulls in their runtime siblings (FreeBSD-pam-dev → FreeBSD-pam
# → FreeBSD-pam-lib) which then race our overlay/build at end
# of build via pkg autoremove.
#
# Nothing in our tree consumes <security/pam_*.h> between pkg
# install and our PAM iter 1: src/OpenPAM uses its own bundled
# openpam/include/security/ via -I; src/pam_modules uses
# -I ../OpenPAM/openpam/include; pamframeworktest +
# pammodulestest build AFTER our OpenPAM `make install` writes
# our headers into the sysroot.
#FreeBSD-pam-dev
FreeBSD-openssl-dev

# Required for libdispatch's compile path: FreeBSD's own
# /usr/include/sys/ucred.h #includes <bsm/audit.h>, so anything that
# transitively pulls <sys/mount.h> needs the audit headers. We curated
# FreeBSD-audit out of the runtime set; the -dev variant lives here so
# headers are present at build time, then purged before mkuzip.
FreeBSD-audit-dev

# in separate pkgbase pkgs — they ship inside FreeBSD-toolchain (which

# `tar -tf FreeBSD-toolchain-*.pkg | grep -E 'yacc|lex|flex|byacc'`
# returning /usr/bin/{yacc,byacc,lex,flex,flex++,lex++}. So no extra
# pkg adds needed for the migcom build.
